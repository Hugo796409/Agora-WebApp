Instructions pour Figma :

Page d’accueil / Landing Page

Titre accrocheur : « Teste ton idée de startup »

Sous-titre : « Apprends à construire, échouer, et rebondir »

Bouton CTA : « Commencer le bac à sable »

UX simple et claire, design jeune, épuré, couleurs dynamiques mais pas flashy

Inscription / Profil (optionnel pour la démo)

Formulaire minimal : nom, prénom, email, âge (18-25)

Pour le prototype : afficher un profil factice ou « utilisateur test »

Accueil bac à sable / Dashboard projet

Vue globale : liste des idées projets (même si fictives pour la démo)

Boutons : « Créer un projet », « Mes projets », « Voir les feedbacks »

Section tutoriel / onboarding rapide expliquant la pédagogie du crash test

Création projet / Build

Étapes interactives :

Nom du projet

Description courte

Choix des ressources / budget fictif

Construction du business plan (formulaire guidé)

Indication visuelle : « barre de progression » ou timeline

Crash Test / Simulation

Message pédagogique : « Testez votre idée à 80% de seuil de réussite »

Feedback immédiat :

Si réussite : encouragement, points d’apprentissage affichés

Si échec : conseils pédagogiques, possibilité de réessayer

Interface simple, style tableau ou carte projet avec indicateurs visuels

Feedback des mentors

Section dédiée au feedback : mentor fictif ou réel

Notes/commentaires affichés sous forme de carte ou pop-up

Option pour voir les points forts et les axes d’amélioration

Fin de session / Récapitulatif

Affichage clair : score global, progrès, compétences acquises

Bouton « Recommencer » ou « Créer un nouveau projet »

Consignes UX/UI

Style épuré, moderne, interface jeune (18-25 ans)

Couleurs : bleu clair / blanc / touches vives pour les CTA

Icônes et illustrations pour rendre la pédagogie visuelle

Mise en avant des messages pédagogiques (feedbacks, encouragements)

Instructions pour Figma / IA design :

Crée un prototype interactif pour le site “Agora – Bac à sable entrepreneurial 18-25 ans”, avec :

Dashboard projet

Formulaire création projet

Simulation crash test avec feedback pédagogique

Feedback mentor affiché sous forme de cartes

Navigation simple entre les étapes (accueil, création, crash test, feedback, récapitulatif)

UX intuitive, couleurs jeunes et modernes, typographie lisible

Ne pas inclure le matchmaking ni les ligues pour cette version

Focus sur apprentissage par l’échec, pédagogie et visualisation du projet
